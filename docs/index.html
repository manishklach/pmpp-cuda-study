Why This Repo

Study CUDA systematically, not as disconnected snippets

The repo is organized as a progression: simple kernels first, then reductions, scans, histograms, dense linear algebra, imaging, simulation, and graph or ML workloads. Examples are intentionally small enough to inspect directly.

The focus is technical study value: correctness before timing, explicit thread and block mapping, memory-behavior notes, synchronization points, and comparisons between baseline and improved kernels where the optimization story is worth learning.

How To Study This Repo

A practical way to use the examples

  1. Run the baseline example first and confirm the CPU reference path is clear.
  2. Check the output and PASS or FAIL result before looking at benchmark numbers.
  3. Inspect thread and block ownership, then note where memory is read, reused, and synchronized.
  4. Compare the improved version against the baseline and ask what changed in data movement or contention.
  5. Use benchmark mode only after you understand why a kernel should be faster.
