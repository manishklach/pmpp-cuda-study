# v0.2.0

Expanded **pmpp-cuda-study** from an initial scaffold-heavy repository into a much more complete PMPP-style CUDA study curriculum.

## Highlights

- Implemented **reference-friendly CUDA examples for `021-100`**, bringing the repository to **96 implemented examples** with only `057-060` remaining as guided templates.
- Added substantial coverage across:
  - reductions, scans, histograms, and sorting primitives
  - matrix, stencil, image, and signal-processing workloads
  - simulation and rendering examples
  - sparse, graph, and ML-oriented study kernels
- Upgraded the top-level README with:
  - an explicit **implementation status** section
  - a **recommended core study path**
  - clearer emphasis on canonical CUDA examples such as vector addition, matrix multiplication, image processing, and reduction
- Added a new **curriculum expansion package** for future growth:
  - `docs/curriculum-map.md`
  - `docs/status.md`
  - reusable templates under `templates/`
  - an updated `tools/generate_examples.py` scaffold utility

## Repository Status In This Release

- `96` examples are marked **Reference-friendly**
- `4` examples remain **Guided template**
  - `057_lu-factorization-sketch`
  - `058_cholesky-factorization`
  - `059_qr-factorization-sketch`
  - `060_fft-based-convolution`

## Notable Additions Since v0.1.0

- `examples/021-040`: implemented parallel patterns such as reductions, scans, compaction, sorting, and selection
- `examples/041-060`: implemented most dense linear algebra and image/filtering examples, with a few advanced factorization/library-heavy topics intentionally left scaffolded
- `examples/061-080`: added image, signal, and simulation-oriented implementations
- `examples/081-100`: added advanced simulation, graph, ML, and all-reduce study examples
- `docs/` and `templates/`: added long-term curriculum planning and scaffolding support for expanding the repo beyond the current 100 examples

## Notes

- This release makes the repository much more usable as a serious self-study curriculum instead of an early-stage scaffold collection.
- The implementation-status wording in the README is now intentionally explicit so readers can distinguish runnable examples from guided templates.
- `nvcc` was not available in the authoring environment used for packaging this release, so CUDA compilation was not executed as part of this release process.
